# Python Sample Projects

Sample projects for Python demonstrating typical CI/CD setup patterns across frameworks, package managers, and version management tools.

## Samples

| Directory | Framework | Package Manager | Testing | Version Management |
|---|---|---|---|---|
| `fastapi-pip-sample` | FastAPI 0.115 | pip + venv | pytest + httpx | `.python-version` |
| `fastapi-uv-sample` | FastAPI 0.115 | uv | pytest + httpx | `.python-version` (uv) |
| `fastapi-poetry-sample` | FastAPI 0.115 | Poetry | pytest + httpx | `.python-version` (pyenv) |

---

### fastapi-pip-sample

**Stack:** FastAPI 0.115 · Python 3.14 · pytest · pip + venv

**What it demonstrates:**
- FastAPI app with root, list, create, get, and delete item routes
- `requirements.txt` / `requirements-dev.txt` split for prod vs dev dependencies
- `requirements.lock` for reproducible installs (generated by `pip freeze`)
- Sync `TestClient` tests with pytest covering happy paths and 404 errors

**How to run:**
```bash
cd fastapi-pip-sample

# Create virtual environment and install dependencies
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

pip install -r requirements-dev.txt

# Run tests
pytest

# Start the development server
uvicorn app.main:app --reload
```

### fastapi-uv-sample

**Stack:** FastAPI 0.115 · Python 3.12 · pytest · uv

**What it demonstrates:**
- Same FastAPI app as `fastapi-pip-sample`
- `pyproject.toml` as the single source of truth for dependencies
- `[dependency-groups]` (PEP 735) for dev dependencies — the uv-native approach
- `uv.lock` for reproducible installs (generated by `uv lock`)
- `pytest` config via `[tool.pytest.ini_options]` in `pyproject.toml`
- `.python-version` for Python version pinning (read by uv automatically)

**How to run:**
```bash
cd fastapi-uv-sample

# Install uv (if not already installed)
curl -LsSf https://astral.sh/uv/install.sh | sh

# Create virtual environment, install all dependencies (including dev group), and generate lock file
uv sync --group dev

# Run tests
uv run pytest

# Start the development server
uv run uvicorn app.main:app --reload
```

### fastapi-poetry-sample

**Stack:** FastAPI 0.115 · Python 3.12 · pytest · Poetry

**What it demonstrates:**
- Same FastAPI app as `fastapi-pip-sample`
- `pyproject.toml` as the single source of truth for dependencies
- `[tool.poetry.group.dev.dependencies]` for dev dependencies — the Poetry-native approach
- `poetry.lock` for reproducible installs (generated by `poetry install`)
- `pytest` config via `[tool.pytest.ini_options]` in `pyproject.toml`
- `.python-version` for pyenv version pinning

**How to run:**
```bash
cd fastapi-poetry-sample

# Install Poetry (if not already installed)
curl -sSL https://install.python-poetry.org | python3 -

# Install all dependencies (creates virtualenv automatically) and generate lock file
poetry install

# Run tests
poetry run pytest

# Start the development server
poetry run uvicorn app.main:app --reload
```
